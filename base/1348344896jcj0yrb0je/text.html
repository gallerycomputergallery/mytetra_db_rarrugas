With OpenMPI

Thread-safe version of OpenMPI must be used!

Load an appropriate module, for example

module load openmpi/1.6.5_mt


FC=ifort CC=icc MPIF90=mpif90 MPIFC=mpif90 ./configure --with-blas="-Wl,--start-group libmkl_intel_ilp64.a libmkl_intel_thread.a libmkl_core.a -Wl,--end-group -openmp" --prefix=/home/yvishnev/Apps/Cfour/cfour_v1_openmpi_mt --enable-mpi=openmpi --with-mpirun="srun -n \$CFOUR_NUM_CORES"


Cfour with IntelMPI it is not working, since binaries are linked with ilp64 version of IntelMPI which is bad for Cfour, since it uses integer(4) everywhere for MPI communacation.


Fix make.config to be sure that full paths to mkl libs exist. The path can be taken from MKLLIB var, which can be checked by 'export -p'.

Remove first '-l' from '-l-Wl,--start-group'.


make

make install
